# Навігатор Markdown NVDA

Навігатор Markdown — це додаток для NVDA, призначений для покращення редагування та читання вмісту Markdown. Він дозволяє використовувати функції навігації за допомогою однієї клавіші, подібні до режиму огляду NVDA, у редагованих текстових областях (таких як Notepad, VS Code, веб-редактори тощо).

Це дозволяє швидко переходити до заголовків, списків, таблиць, блоків коду та різних вбудованих форматувань без необхідності часто натискати клавіші зі стрілками.

## Основні особливості

- **Ефективна навігація**: реалізує швидкі алгоритми, що дозволяють миттєво переходити навіть у великих документах, що містять десятки тисяч рядків.
- **Структурований перегляд**: підтримує навігацію за заголовками, списками, таблицями, цитатами, блоками коду тощо.
* **Вбудовані елементи**: підтримує перехід до жирного шрифту, курсиву, посилань, зображень та вбудованого коду.
* **Навігація по комірках таблиці**: надає скорочення, сумісні з NVDA, для переміщення між комірками в таблицях Markdown.

## Використання

### Перемикання режиму огляду
Початково навігація за допомогою однієї клавіші вимкнена. Вам потрібно вручну увімкнути режим огляду Markdown.

* **Комбінація клавіш**: `NVDA + Shift + Пробіл` (пробіл)
* **Ефект**: вмикає або вимикає функції додатка. NVDA оголосить «Увімкнення/вимкнення режиму огляду Markdown». Ця функція також відповідає налаштуванню NVDA «Звукові індикації для режимів фокусування та огляду» в налаштуваннях режиму огляду.

### Список комбінацій (якщо увімкнено режим огляду)

Після увімкнення цього режиму ви можете використовувати наступні команди.
* Натисніть **літеру**, щоб перейти до наступного елемента.
* Натисніть **Shift + літеру**, щоб перейти до попереднього елемента.

| Клавіша | Тип елемента | Опис |
| :--- | :--- | :--- |
| **H** | Заголовок | Заголовок будь-якого рівня (від `#` до `######`) |
| **1-6** | Заголовок 1-6 | Конкретний рівень заголовка |
| **T** | Таблиця | Початок таблиці |
| **L** | Список | Початок блоку списку |
| **I** | Елемент списку | Конкретний елемент списку (`-`, `*`, `1.`) |
| **Q** | Цитата | Блок цитати (`>`) |
| **C** | Код | Блок коду (```` ``` ````) або вбудований код (`` ` ``) |
| **S** | Роздільник | Горизонтальна лінія (`---`, `***`) |
| **X** | Прапорець | Елемент списку завдань (`- [ ]`, `- [x]`) |
| **K** | Посилання | Посилання Markdown `[текст](url)` |
| **G** | Графіка | Тег зображення `![alt](url)` |
| **B** | Жирний шрифт | Жирний текст (`**` або `__`) |
| **E** | Виділення | Курсив/виділений текст (`*` або `_`) |
| **D** | Видалити | Перекреслений текст (`~~`) |
| **F** | Виноска | Посилання на виноску `[^1]` |
| **,** | Кінець блоку | Перехід до кінця поточного блокового елемента |
| **Shift+,**| Початок блоку | Перехід до початку поточного блокового елемента |
| **M** | Математика | Математичний вираз LaTeX (`$...$` або `$$...$$`) |

### Навігація комірками таблиці

Коли курсор знаходиться всередині таблиці Markdown і ввімкнено режим огляду, ви можете використовувати такі комбінації клавіш для переходу між комірками:

- **Ctrl + Alt + стрілка вліво**: попередня комірка
- **Ctrl + Alt + стрілка вправо**: наступна комірка
- **Ctrl + Alt + стрілка вгору**: комірка вище
- **Ctrl + Alt + стрілка вниз**: комірка нижче

## Примітки

- Цей додаток спеціально розроблений для редагування Markdown. Якщо ви читаєте HTML-сторінку, відтворену браузером, використовуйте вбудований режим огляду NVDA.
* Якщо натискання клавіш призводить до введення символів замість навігації, перевірте, чи увімкнено режим огляду, натиснувши `NVDA + Shift + Пробіл`.
* У VS Code, через те, що редактор відображає вміст за запитом (віртуалізація), режим огляду цього додатка може не працювати як очікується для вмісту, який наразі не показується/не візуалізується.
